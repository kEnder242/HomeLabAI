# Remote Access Setup Guide
# =========================

You are receiving this because you are setting up remote access to the Acme Lab (z87-Linux).

## 1. SSH Access (Pre-Cloudflare)
If you set up Tailscale, you can SSH directly:
  ssh jallred@z87-Linux (or IP)

## 2. Cloudflare Tunnel Setup (Interactive)
Since I could not authenticate for you, you must do this step interactively on the Linux machine.

1.  **Authenticate:**
    Run this command and copy the URL to your browser (on any device) to authorize 'acme-lab':
    `cloudflared tunnel login`

2.  **Create the Tunnel:**
    `cloudflared tunnel create acme-lab`
    (Copy the Tunnel ID formatted UUID, e.g., f8942-...)

3.  **Configure Routing (DNS):**
    Choose a subdomain you own (e.g., lab.yourdomain.com).
    `cloudflared tunnel route dns acme-lab lab.yourdomain.com`

4.  **Create Config File:**
    Edit `~/.cloudflared/config.yml`:
    ```yaml
    tunnel: <Tunnel-UUID>
    credentials-file: /home/jallred/.cloudflared/<Tunnel-UUID>.json

    ingress:
      - hostname: code.yourdomain.com
        service: http://localhost:8080
      - hostname: pager.yourdomain.com
        service: http://localhost:8501
      - service: http_status:404
    ```

5.  **Start the Tunnel:**
    `cloudflared tunnel run acme-lab`

## 3. Starting the Services

**Service A: Code-Server (VS Code)**
Run this in a tmux session or background:
  `code-server --bind-addr 127.0.0.1:8080 --auth password`
  (Check `~/.config/code-server/config.yaml` for the password)

**Service B: The Pager (Streamlit)**
Run this in a tmux session:
  `cd ~/AcmeLab`
  `.venv/bin/streamlit run src/pager.py`

## 4. Access
Open your browser to:
- https://code.yourdomain.com (Password from config)
- https://pager.yourdomain.com (The Pager UI)
